# 负荷预测（电力调度基础）— 电力交易

> 作者：王立讲电力交易
> 时长：约6分26秒
> 链接：[https://v.douyin.com/4EymsRPbUDc/](https://v.douyin.com/4EymsRPbUDc/)

***

## 核心观点

负荷预测是电力调度和电力交易的基础。调度并不关心具体某个用户的用电负荷，它的要求是**总发电等于总用电（总负荷）**。负荷预测的核心任务就是回答两个问题：**预测多少负荷？在哪个节点消耗？** 整个预测流程看似高大上，难点其实不在构建模型，而在前期的数据收集与预处理——找到与负荷相关的影响因素并数据化呈现。

***

## 一、预测什么负荷？

### 1. 系统总负荷

整个系统内所有用电负荷曲线的加合。调度关心的是总发电 = 总用电。

### 2. 母线负荷

母线是电网中接收和分配电能的核心节点，所有发电机、负荷、输电线都通过母线连接。预测母线负荷本质上是**预测每个关键节点的具体用电需求**。

> 负荷具有空间性质——大家可以在各地用电，所以需要预测系统总负荷 + 母线负荷。

### 三类典型用户的负荷曲线

| **用户类型**     | **特征**       | **典型曲线**                          |
| ------------ | ------------ | --------------------------------- |
| **工业用户**     | 连续生产，24小时不断电 | 24小时平稳，无大幅波动                      |
| **商业用户（商场）** | 跟营业时间强相关     | 开业前低位 → 开业爬升 → 中午稳定 → 晚间高峰 → 关业回落 |
| **居民用户**     | 早高峰 + 晚高峰双峰  | 早起高峰 → 中午低位 → 下午爬升 → 晚高峰 → 深夜低位   |

***

## 二、负荷预测的三个周期

### 1. 中长期负荷预测

| **属性** | **说明**         |
| ------ | -------------- |
| 周期     | 月、年、数十年        |
| 目标     | 安排检修计划、支撑电网规划  |
| 数据要求   | 至少5年的年度或月度负荷数据 |

**影响因素**：

* 宏观经济：GDP增速、三大产业、工业增加值
* 社会因素：人口总量与分布、城镇化率
* 特殊季节：采暖度日、制冷度日
* 政策与规划：产业政策、能源政策

### 2. 短期负荷预测

| **属性**   | **说明**           |
| -------- | ---------------- |
| 周期       | 1日到1周            |
| 颗粒度      | 15分钟（一天96个点）     |
| 目标       | 制定次日发电计划（日前发电计划） |
| **精度要求 | 97%**          |

**影响因素**：

* 历史负荷数据（近1-3年同日/同类型日）
* 气象数据（次日温度、湿度、降水、风速）
* 日期类型（工作日/节假日）
* 重点工业用户生产计划
* 居民用电习惯变化（如电动汽车充电）

### 3. 超短期负荷预测

| **属性**   | **说明**      |
| -------- | ----------- |
| 周期       | 分钟级到小时级     |
| 颗粒度      | 5分钟或15分钟    |
| 目标       | 制定日内和实时发电计划 |
| **精度要求 | 99%**     |

**影响因素**：

* 实时负荷数据（最近10-30分钟）
* 实时气象数据（当前温度、风速、光照强度）
* 实时设备数据（电网大型用户实时用电状态）
* 新能源电站实时出力数据

### 三个周期的递进关系

```
中长期 → 短期 → 超短期
规划期 → 日前 → 实时
精度递增：97% → 99%
时效性递增
```

***

## 三、负荷预测的五个步骤

### Step 1：明确目标和边界

* 确定预测范围：预测什么（量/日/实时/月度）
* 明确空间范围：北京市 / 山西省 / 山东省 / 整个华北区域
* 设定精度要求：97% 还是 99%
* 定义输出形式：颗粒度是什么

### Step 2：数据收集与预处理

* 收集相关数据（历年/日/实时）
* 数据预处理：异常值处理、归一化、特征工程

### Step 3：选择与构建预测模型

* 模型选择：回归、时间序列、ARIMA、指数平滑等
* 构建与训练：划分数据集、参数调优

### Step 4：执行预测与结果输出

* 模型预测输出
* 不确定性分析（置信区间）

### Step 5：预测结果校验与修正

* 误差计算：MAPE、RMSE 等指标
* 误差分析 → 修正模型 → 迭代优化

***

## 核心洞察

> 负荷预测看起来很高大上，其实也不过就是这五个步骤。难点并不在所谓的构建模型，而是在**前期的收集与预处理**——找到与负荷相关的影响因素有哪些，你是否能把它数据化呈现。

> 你去猜明天的用电行为，是不是可以参考今天的用电行为？最简单的方法就是用**昨天的数据预测今天**。

***

## 思考与延伸

1. **电动汽车对负荷曲线的重塑**：居民用电从"生活用电"变成"生活用电 + 电动汽车充电"，晚间的负荷曲线将被显著改变，这对短期负荷预测的模型输入提出了新要求。
2. **新能源并网增加预测难度**：风电、光伏的出力具有强随机性，超短期负荷预测需要同时考虑负荷侧和电源侧的双重不确定性。
3. **精度要求的现实意义**：短期97%、超短期99%——看似只差2个百分点，但在省级电网的负荷量级下，2%可能意味着几十万千瓦的偏差，直接影响发电计划的执行质量。
4. **母线负荷预测对电力交易的价值**：在现货市场环境下，不同节点的电价不同，母线负荷预测直接影响节点电价的预测精度，进而影响交易策略。
5. **数据是负荷预测的核心资产**：历史数据的积累、气象数据的接入、用户行为数据的获取——谁的数据质量高，谁的预测精度就高。

## 完整文稿

> 首先负荷预测，预测什么负荷？用电行为是一个高度随机性和分散性的。我们看三类用户典型的日负荷曲线。

> 第一类用户是工业用户，连续生产性的，24小时不断电，24小时无停生产。

> 第二个曲线是我们典型的商场，商业用户的曲线。到它开业之前一般是上午8点9点开业，那晚上这一段时间它会有一些基础负荷，比如说指示灯亮灯，然后逐步去爬升，中午下午是一个稳定运行的阶段，到了晚上来商场的人又多了，用电又会上一些，最后直到9点10点商场歇业，用电负荷又降下来，回到一个低负荷的状态。

> 第三个是居民用户，很明显晚上不怎么用电，然后早上起床有一个早高峰，中午用得比较少，下午逐步升高直到晚高峰，两个高峰然后再下降。

> 调度其实并不关心具体某个用户的用电负荷，它的要求是总发电等于总用电（总负荷）。所以第一个我们要预测的是整个系统内所有用电负荷曲线的加合，就是我们说的系统总负荷。

> 第二个，有了总量够吗？其实还不够，因为需要知道电能具体在哪消耗。母线是电网中接收和分配电能的核心节点，所有发电机、负荷、输电线都需要通过母线连接。预测母线负荷本质上是预测每个关键节点的具体用电需求。负荷具有空间性质，大家可以在各地用电，所以我们要预测系统总负荷和母线负荷。

> 负荷预测有哪些周期？三个周期：中长期负荷预测、短期负荷预测、超短期负荷预测。

> 中长期负荷预测周期一般是月、年、数十年，目标是安排检修计划以及支撑电网规划。影响因素包括宏观经济GDP增速、三大产业工业增加值、社会因素人口总量与分布、城镇化率、特殊季节采暖度日制冷度日、政策与规划产业政策能源政策。做中长期预测至少要五年的年度或月度负荷数据。

> 短期负荷预测周期是一日到一周，颗粒度是15分钟（一天96点），目标是制定次日发电计划（日前发电计划），精度要求是97%。影响因素包括历史负荷数据近一到三年的同日同类型日负荷数据、气象数据次日的温度湿度降水风速、日期类型工作日节假日、重点工业用户生产计划、居民用电习惯变化（如电动汽车充电）。

> 超短期负荷预测周期是分钟级到小时级，颗粒度5分钟或15分钟，目标是制定日内和实时发电计划，精度要求99%。影响因素包括实时负荷数据最近10到30分钟、实时气象数据当前温度风速光照强度、实时设备数据电网大型用户实时用电状态、新能源电站实时出力数据。

> 负荷预测的五个步骤：明确目标和边界、数据收集与预处理、选择与构建预测模型、执行预测与结果输出、预测结果校验与修正。

> 难点并不在构建模型，而是在前期的收集与预处理——找到与负荷相关的影响因素有哪些，你是否能把它数据化呈现。
