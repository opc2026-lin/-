### 🧠 Karpathy LLM+Wiki 外脑系统规则

> 核心理念：**LLM 是全职图书管理员，人类只负责喂原料和提问题。**
> 数据主权：**File over App** — 一切是本地 Markdown，永不锁定。

#### 三层架构

| 层级 | 目录 | 职责 |
|------|------|------|
| **原料层** | `收件箱/` | 不可变原始素材（视频笔记、网页剪藏、IMA 导入）。只追加，不修改 |
| **知识层** | `综合能源/` `能源费用托管/` `售电业务/` `亿云能源/` `交易策略/` `AI知识库/` `个人笔记/` | 按领域分目录，LLM 维护的结构化 Wiki。人类不手动编辑，AI 负责创建/更新/交叉引用 |
| **规则层** | `System/` | 外脑宪法。Schema 定义、实体模板、Lint 规则、工作流定义 |
| **方法层** | `Insights/` | 可复用的思维框架、操作流程、最佳实践 |
| **归档层** | `已归档/` | 不再活跃的历史内容 |

#### 三个核心操作

- **① Ingest（摄入）**：原料 → 按领域路由到对应文件夹。LLM 读取收件箱，判断内容归属，放入正确的知识库目录。一次摄入可能触发 8-15 个关联页面的级联更新。检测与已有论断的冲突并标注分歧。

#### Ingest 路由表

| 内容类型 | 目标目录 | 示例 |
|---------|---------|------|
| 能源托管政策/案例/合同 | `综合能源/` | 政策文件、项目方案、节能改造案例 |
| 能源费用托管课程资料 | `能源费用托管/` | 张乐共享课程、招标文件、医院案例 |
| 售电/电力交易 | `售电业务/` | 电价政策、交易策略、客户清单 |
| 公司业务/内部资料 | `亿云能源/` | 公司方案、培训材料、客户文档 |
| 金融交易/投资复盘 | `交易策略/` | 股票/基金/加密货币交易记录 |
| AI 工具/技术/编程 | `AI知识库/` | AI Skill、Claude Code、Vibe Coding、开发工具 |
| 个人思考/日记/杂感 | `个人笔记/` | 日常记录、想法碎片、非领域内容 |
| 方法论/工作流/原则 | `Insights/` | 可复用的思维框架、操作流程、最佳实践 |
| 已完成/过时/废弃 | `已归档/` | 不再活跃的历史内容 |
- **② Query（查询）**：提问 → 综合。LLM 先读 index.md 定位相关页面，沿 wiki 链接拉取上下文，综合回答并附引用链接。好答案归档回知识库 → 知识滚雪球。
- **③ Lint（巡检）**：自动健康检查。扫描断链、索引一致性、孤立页面、超过 90 天未更新的陈旧内容、缺失的交叉引用。

#### 核心原则

1. **摘要而非切片**：不让 LLM 检索原始文档的向量块，而是让 LLM 一次性编译成结构化 Wiki，然后持续维护
2. **级联更新**：一个新材料入库，不止创建一个页面，而是更新所有相关的概念页、实体页、索引页
3. **冲突标注**：新信息与已有知识矛盾时，不覆盖、不盲信，标注分歧等待人工裁决
4. **知识滚雪球**：每次查询的好答案归档为知识 → 下次查询更聪明
5. **不留孤儿**：每篇笔记必须有 [[wiki链接]]，每条链接必须双向可追踪
6. **超过 100 篇/40 万字时启用分层索引**：现代上下文窗口是主力，向量搜索是后备

#### 文件命名规范

- 原料：`收件箱/{类型}_{关键词}_{ID}.md`
- 知识：`AI知识库/{概念名}.md`（一个概念一个文件，长文不拆分）
- 规则：`System/{领域}/{文件名}.md`

#### 入口

- 将 `AI知识库/` 作为唯一入口
- 将 `System/` 作为规则中心
- 将 `Insights/` 作为方法沉淀中心
