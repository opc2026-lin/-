## 二、笔记格式规范

每篇笔记必须包含：
1. **YAML frontmatter** — `tags:` 标签数组
2. **来源引用** — 标注出处（IMA 笔记 / 视频 / 网页 / 原创）
3. **双向链接** — `[[相关笔记]]` 至少 2 条
4. **段落层级** — 用 `##` `###` 组织，不超过 3 级

## 四、三个核心操作

### Ingest（摄入）
输入：收件箱中的新文件
动作（按顺序自动执行）：
1. 读取文件内容
2. 判断归属目录（见路由表）
3. 移动到目标目录
4. 打上标签 `bash: python3 tag_notes.py`
5. 建立双链 `bash: python3 -c "..."` (基于标签相似度)
6. 更新目标目录的 index.md
7. 刷新仪表盘 `python3 refresh-dashboard.py`
8. Git 提交 `git add -A && git commit -m "Ingest: ..." && git push`

### Query（查询）
输入：用户问题
动作：
1. 读 index.md 定位相关目录
2. 按标签筛选候选笔记
3. 沿 [[wiki链接]] 拉取关联上下文
4. 综合回答，附 `[[笔记名]]` 引用
5. 好答案归档为笔记

### Lint（巡检）
每月 1 日自动执行：
1. 扫描断链 `[[不存在的笔记]]`
2. 检查孤立笔记（入链=0 且 出链=0）
3. 标记陈旧内容（>90 天未更新）
4. 检查 index.md 与目录一致性

## 五、文件命名规范

- 原料：`收件箱/{类型}_{关键词}.md`
- 知识：`{目录}/{核心主题}.md`

## 六、Wiki 页面模板

### 概念页

```markdown
---

---
# 概念名

> 来源: xxx

## 定义
一句话定义

## 核心要点
- 要点1
- 要点2

## 相关笔记
- [[关联笔记1]]
```

### 实体页（人/组织/项目）
```markdown
---
tags: [实体类型]
---
# 实体名

| 属性 | 值 |
|------|-----|
| 类型 | 人/组织/项目 |

## 关联笔记
- [[涉及该实体的笔记]]
```
