# 能源服务公司业财一体化系统 · 需求分析与建设方案

> 编制日期：2026年6月
> 适用范围：福建省能源服务有限公司（电力运维/维修/试验、售电、工程、系统服务）

***

## 一、公司概况

* 主营业务：电力运维、维修、试验、售电、工程施工、系统服务
* 财务人员：3-4人
* 月均工单：40-50个
* IT资源：有阿里云账号
* 现状：使用财小白（外包财务系统）+ 企业微信审批
* 核心诉求：替代财小白，自建全链条业财一体化系统

***

## 二、痛点和需求全景

### 2.1 业务管理类

| **#** | **痛点**      | **具体表现**                        | **解决方向**        |
| ----- | ----------- | ------------------------------- | --------------- |
| 01    | 业务环节衔接脱节    | 合同→采购→施工→开票→回款，各环节靠人喊人，信息传递丢失   | 工作流引擎自动流转+通知    |
| 02    | 维修小单杂、无合同   | 很多维修单没签合同，是否开票、是否回款无法跟踪         | 工单系统，无合同也能入应收   |
| 03    | 开票等业主确认，回款慢 | 不是按合同节点开票，而是等客户说"可以开了"才开，回款被动滞后 | 合同付款计划+自动催收提醒   |
| 04    | 项目完工状态不闭环   | 活干完了但工程部没点完工，财务不知道能否开票          | 手机端强制完工确认+超时提醒  |
| 05    | 施工前内容不清晰    | 工作内容/维修改造内容/注意事项不明确，导致返工浪费人工    | 施工任务单推送到手机端     |
| 06    | 现场勘察数据不齐    | 勘察数据不全，采购买不到适用的产品               | 标准化勘察单+强制填写     |
| 07    | 客服部审批抄送看不到  | 开票/合同审批里抄送了客服部，但他们收不到通知         | 审批抄送+企微自动通知     |
| 08    | 新员工不知道公司业绩  | 不了解公司做过什么项目，面对客户没底气             | 客户/项目档案知识库      |
| 09    | 业务合同备案看不到   | 业务在企微审批完合同，其他人不知道也找不到           | 统一合同台账+审批通过自动通知 |
| 10    | 同一项目重复提交    | 补材料就重新提交，产生多个记录，财务不知道用哪个        | 一项目一合同版本管理      |

### 2.2 财务核算类

| **#** | **痛点**      | **具体表现**                         | **解决方向**         |
| ----- | ----------- | -------------------------------- | ---------------- |
| 11    | 服务类项目成本核算难  | 人工成本、零星材料无法自动归集到项目               | 工时系统+自动分摊引擎      |
| 12    | 三类业务核算规则不同  | 工程施工（到账确认）、服务类（摊销）、售电（开票确认），规则各异 | 按业务类型配置核算规则      |
| 13    | 材料成本归集不清    | 能区分项目的材料直接归集，不能区分的零星材料无分摊依据      | 零星材料按工时分摊        |
| 14    | 内部人工分摊复杂    | 售电人员固定比例、孟超项目单独核算、其余按工时分摊        | 员工分摊规则配置表        |
| 15    | 合同金额变动不通知财务 | 业务口头变更金额，不走流程，财务不知情，月底对不上        | 合同变更审批+强制流程      |
| 16    | 售电渠道费用录入繁琐  | 系统中字段设得太细，回款明细不该设必填              | 取消回款明细必填，Excel导入 |
| 17    | 采购不关联项目     | 采购单没有关联到项目/工单，库存入库和成本归集都出问题      | 采购单强制关联项目        |
| 18    | 设备采购状态不明    | 货到没到、到哪了、谁签收的，全不知道               | 采购跟踪看板           |

### 2.3 审批流程类

| **#** | **痛点**     | **具体表现**                     | **解决方向**          |
| ----- | ---------- | ---------------------------- | ----------------- |
| 19    | 审批与财务脱节    | 企微审批过了，财小白没数据，财务要手工录         | 审批结果自动触发业务/财务数据变更 |
| 20    | 企微审批模板过多   | 约28个审批模板，同类审批拆了多个模板，管理混乱     | 合并为6个通用审批         |
| 21    | 客户/项目名称不完整 | 业务提交合同时客户名、项目名写不全，导致后续所有环节出错 | 强制下拉选客户+名称校验      |
| 22    | 开票需等待通知    | 完工后财务不能直接开票，要等业务口头说"可以开了"    | 业务手机端标记"可开票"      |
| 23    | 抄送功能不可见    | 客服部在审批中被抄送但收不到查看通知           | 审批抄送自动推送企微消息      |

### 2.4 管理与报表类

| **#** | **痛点**     | **具体表现**                   | **解决方向**  |
| ----- | ---------- | -------------------------- | --------- |
| 24    | 项目无全生命周期管理 | 项目走到哪步了不知道，到处问人            | 项目中心一页看全  |
| 25    | 老板看不到实时数据  | 财务报表严重滞后，不知道本月赚/亏、现金流够不够   | 实时手机看板    |
| 26    | 合同内服务项目遗漏  | 现场沟通口径与合同不一致，验收不通过，财务付款出问题 | 服务清单与合同对账 |

***

## 三、系统整体设计

### 3.1 系统定位

自建业财一体化管理系统，替代财小白，覆盖从合同到回款再到项目利润的全链条。

### 3.2 技术栈

| **层** | **技术**          | **说明**      |
| ----- | --------------- | ----------- |
| 后端    | Python Flask    | 轻量、好维护      |
| 数据库   | PostgreSQL      | 阿里云RDS      |
| 前端PC  | 简页面             | 财务/采购/管理后台  |
| 手机端   | 微信小程序           | 维修员现场使用     |
| 消息通知  | 企业微信消息          | 审批待办/催收/通知  |
| OCR   | PaddleOCR（免费开源） | 拍照合同自动识别    |
| 部署    | 阿里云ECS轻量服务器     | 2核4G约500元/年 |

### 3.3 系统架构（七层）

```
┌─────────────────────────────────────────────┐
│              用户访问层                        │
│   PC管理后台  |  微信小程序  |  手机浏览器    │
├─────────────────────────────────────────────┤
│              外部对接层                        │
│   企业微信(消息)  |  银行(流水)  |  开票平台  │
├─────────────────────────────────────────────┤
│              报表输出层                        │
│   三大财报 | 项目利润表 | 应收账龄 | 业务线毛利 │
├─────────────────────────────────────────────┤
│              核心财务层                        │
│   总账 | 费用管理 | 资产管理 | 薪酬 | 现金流   │
├─────────────────────────────────────────────┤
│              业财桥梁层                        │
│   开票→应收 | 回款核销 | 项目成本中心 | 应付   │
├─────────────────────────────────────────────┤
│              业务流转层                        │
│   合同管理 | 采购管理 | 库存管理 | 工单/派工   │
├─────────────────────────────────────────────┤
│              基础数据层                        │
│   客户 | 物料 | 员工 | 供应商 | 会计科目      │
└─────────────────────────────────────────────┘
```

## 四、三大业务模块核算规则

### 4.1 工程施工

| **项目** | **规则**    | **系统实现**      |
| ------ | --------- | ------------- |
| 收入确认   | 实际工程款到账时点 | 银行回单匹配合同→自动确认 |
| 材料成本   | 当月实际领料出库  | 出库单关联工单→自动结转  |
| 人工+费用  | 实际支付时点    | 付款单关联合同→自动结转  |

### 4.2 服务类（运维/试验/维修）

| **项目**  | **规则**              | **系统实现**            |
| ------- | ------------------- | ------------------- |
| 收入确认    | 合同金额按月摊销            | 每月1号自动摊销（合同金额÷合同月数） |
| 材料成本    | 能区分项目→直接归集；零星→按工时分摊 | 出库单直接归集+月底按工时比例分摊   |
| 内部人工    | 孟超/售电单独核算；其余按工时比例分摊 | 员工分摊规则配置+月底自动分摊     |
| 外协人工+费用 | 实际支付时结转             | 付款单关联合同→自动结转        |

### 4.3 售电

| **项目** | **规则**             | **系统实现**                         |
| ------ | ------------------ | -------------------------------- |
| 收入确认   | 当月开票金额确认           | 开票事件触发→自动确认                      |
| 人工成本   | 固定比例分摊             | 叶凡100%、郑章文100%、林总50%、陈姐20%、柳娟20% |
| 费用     | 渠道费等实际支付+总部运营费5%分摊 | 付款单+月底自动分摊                       |
| 材料成本   | 无                  | —                                |

## 五、审批流程设计

### 5.1 合并方案

| **新系统审批模板**  | **合并原企微模板**               | **审批通过后自动触发**     |
| ------------ | ------------------------- | ----------------- |
| ① 业务合同审批（通用） | 售电/运维/工程/试验/维修/系统安装合同（6个） | 合同生效→生成应收计划→通知可开工 |
| ② 抢修/临时派工备案  | 抢修/维修/临时派工备案（1个）          | 生成紧急工单→派工到手机端     |
| ③ 采购审批（统一）   | 备件申请/需求申请/合同备案（3个）        | 生成采购单→采购员可下单      |
| ④ 开票申请       | 开票申请/二次开票（1个）             | 生成应收记录→通知财务开票     |
| ⑤ 付款申请（通用）   | 物资付款/保证金/其他付款（多个）         | 生成付款单→冲减应付        |
| ⑥ 费用报销（通用）   | 费用报销/商务费/管理费用/挂靠结算（多个）    | 生成凭证→归集到项目成本      |

**留企微的**：用章申请、请假、试验用章（与财务无关，不动）

### 5.2 审批与财务的联动

```
审批通过
  ↓
系统自动执行（无需人工介入）：
  合同审批→合同状态变"已生效"+应收计划生成
  采购审批→采购单可下单
  开票审批→通知财务开票+应收生成
  付款审批→生成付款单+冲减应付
  报销审批→生成凭证+费用归集到项目
  ↓
推送企微通知给相关人员
```

### 5.3 审批中的强制规则

* 客户名称：只能从客户表下拉选择，不允许手写
* 采购单：必关联项目/工单（备库除外）
* 合同金额变更：必须走"合同变更"流程
* 开票金额超合同：系统自动拦截，提示先办变更

## 六、关键功能设计

### 6.1 OCR拍照录入

```
业务员拍合同/单据
  ↓
PaddleOCR识别（客户名/金额/日期/内容）
  ↓
自动填入表单（客户名匹配到系统客户表）
  ↓
业务员逐项检查→修改→点"确认无误"
  ↓
提交审批/生效
```

### 6.2 手机端（微信小程序）

维修员能用手机完成：

* 查看今日派工
* 开始/结束工单
* 填工时、选物料（搜索名称，不用编码）
* 拍完工照片
* 客户手机签字
* 标记"可开票"

### 6.3 项目成本中心

```
项目收入
  - 物料成本（出库单归集+零星分摊）
  - 人工成本（工时×费率+固定比例分摊）
  - 费用成本（报销关联项目）
  - 税金
  = 项目净利润
```

### 6.4 老板实时看板

手机端能看到的：

* 本月收入/成本/利润（实时）
* 应收总额+账龄分布
* 各业务线毛利
* 未来30天回款预测
* 项目利润排行

## 七、实施路线（四期）

### 第1期（1-2周）：工单→应收→回款

| **模块** | **功能**          | **解决痛点**   |
| ------ | --------------- | ---------- |
| 基础数据   | 客户/物料/员工主数据     | 统一数据规范     |
| 工单管理   | 派工→完工→客户签字（手机端） | 维修单跟踪、完工闭环 |
| 开票管理   | 发票录入→自动生成应收     | 开票跟踪       |
| 回款核销   | 逐笔勾稽+先进先出自动匹配   | 回款跟踪       |
| 应收报表   | 客户应收账龄表         | 老板看到欠款情况   |

### 第2期（3-4周）：采购→库存→成本

| **模块** | **功能**         | **解决痛点**   |
| ------ | -------------- | ---------- |
| 采购管理   | 请购→比价→下单→到货→退货 | 采购状态透明     |
| 库存管理   | 入库→出库→退库→盘点→预警 | 库存实时可知     |
| 手机端    | 工时记录+用料选择上线    | 成本初步归集     |
| 应付账款   | 采购入库→应付→付款     | 欠供应商多少一目了然 |

### 第3期（2个月）：凭证自动+项目利润

| **模块** | **功能**          | **解决痛点**    |
| ------ | --------------- | ----------- |
| 成本中心   | 物料+人工+费用+税金四维归集 | 终于知道每个项目赚多少 |
| 凭证自动   | 业务事件→自动生成凭证     | 财务不再手录      |
| 费用管理   | 报销+关联项目+分摊      | 费用归集到项目     |
| 项目利润表  | 收入-成本=净利润       | 老板决策依据      |
| 合同变更   | 金额变更审批+自动调整     | 不让财务被蒙在鼓里   |

### 第4期（持续迭代）

* 资产管理（固定资产登记+折旧）
* 薪酬核算（工资+社保+分摊）
* 银企直连（流水自动同步）
* 税务申报底稿（增值税/所得税导出）
* 现金流预测（未来30/60/90天）
* 关账流程（月结检查清单）
* 三大财报自动生成
