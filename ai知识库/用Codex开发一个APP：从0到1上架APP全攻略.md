# 用Codex开发一个APP：从0到1上架APP全攻略

> 作者：Lala罐头
> 时长：约7分35秒
> 链接：[https://v.douyin.com/c5F-4B\_RKl0/](https://v.douyin.com/c5F-4B_RKl0/)

***

## 核心观点

不到一年时间，Vibe Coding 发生了质变——AI 不光会写代码，还会自己跑测试、自己截图找 bug、自己操作电脑帮你打包上架。作者作为文科生，用 Codex 从零开发了一款 AI 美甲 APP 并成功上架 App Store，完整演示了 Vibe Coding 2.0 的全流程。

***

## Vibe Coding 1.0 vs 2.0

| **维度** | **1.0（去年）**  | **2.0（现在）**                |
| ------ | ------------ | -------------------------- |
| AI 能力  | 能跑完开发但经常中断出错 | 会写代码、跑测试、截图找 bug、操作电脑、打包上架 |
| 操作复杂度  | 在 IDE 上来回折腾  | Codex 界面极简，只有项目栏 + 对话框     |
| 适合人群   | 需要懂编程        | 文科生也能用                     |

***

## 前置准备：安装插件

进入 Codex 插件中心，安装以下插件：

| **插件**           | **功能**              | **角色** |
| ---------------- | ------------------- | ------ |
| **Superpower**   | 全面的项目管理，把模糊想法拆成开发计划 | 产品经理   |
| **Computer Use** | AI 直接操作你的电脑         | 执行者    |
| **Chrome**       | AI 在浏览器里操作          | 浏览器自动化 |
| **Builds Apps**  | 在模拟器里预览 APP 效果和测试   | 测试员    |

***

## 全流程六步法

### Step 1：创建项目 + 选模型

* 创建 APP 项目文件夹（后续文件统一放在项目夹里）
* 选择 **o3-mini 模型** + **中度推理模式**
* 速度和结果平衡较好

### Step 2：需求拆解（Superpower 插件）

**新手常见误区**：一上来就想要 AI 一句话生成好一个 APP → 代码又多又长、额度消耗大、跑半天不知道进度。

**正确做法**：用 Superpower 插件做需求拆解。

**操作**：点击 Superpower → 输入提示词：

> "我要开发一款 AI 美甲的产品，并最终打包发布在 App Store 上面"

**Superpower 的工作流程**：

1. **Brainstorming（脑暴）**：跟你有来有回地讨论
2. **选择题确认**：逐步确认核心目标、实现路径等
   * 核心目标：给普通美甲用户做试戴
   * 实现路径：每次 AI 生成新图片（而非套模板）
3. **输出完整开发计划**：包括流程设计 + 技术路线设计

**开发计划内容**：

* 流程设计
* 技术路线（Expo 等）
* 核心功能（数据接口、测试验收等）
* 相当于"为你而生的开发地图，顺着走就行"

### Step 3：前端视觉对齐（写代码前！）

**核心原则**：在 AI 写任何一行代码之前，先把前端页面视觉对齐。

> 代码你看不懂，文字又容易有歧义，**图片是最高效的对齐方式**。

**操作**：

1. 让 AI 生成前端可视化展示
2. 可以指定参考风格和样式
3. 反复微调（如 logo 区域压低、留白增多、手的位置比例调整）
4. 基于主页图生成其他接触页面（设置页、功能页等）
5. 直接在页面上调整（如把会员订阅改成积分方式）

**产出**：5-6 个页面的完整前端视觉稿

### Step 4：前端代码生成 + 模拟器测试

**操作**：

1. 让 AI 基于确认的前端图生成页面代码
2. 等待约 **44 分钟**生成前端交互页面
3. 用 **Builds Apps 插件**在模拟器里看实时效果
4. 在模拟器上验证交互功能
5. 发现问题（按钮无反应、按钮不对等）→ 截图发给 Codex → 快速修改

**来回几趟修改后，前端交互没问题。**

### Step 5：后端代码开发

**操作**：

1. 让 AI 继续生成后端代码
2. 同步开启 **Plan Mode**，了解 AI 每一步在做什么
3. AI 会给出完整的任务计划
4. 逐个实现细小功能：上传图片、AI 生图、积分功能等

### Step 6：打包上架

**遇到的问题**：开发到这个阶段，大概率会遇到同一个问题——**打包失败**。

**解决方案**：

* 用 **Computer Use 插件**让 AI 直接操作电脑
* AI 自己打开终端、运行命令、排查错误、修复问题
* 最终完成打包并上架 App Store

## 关键心法

| **心法**                 | **说明**                   |
| ---------------------- | ------------------------ |
| **别一句话让 AI 生成整个 APP**  | 代码又多又长，额度消耗大，进度不可控       |
| **写代码前先对齐视觉**          | 图片是最高效的对齐方式，代码看不懂、文字有歧义  |
| **用 Superpower 做产品经理** | 把模糊想法拆成结构化开发计划           |
| **Plan Mode 同步开启**     | 了解 AI 每一步在做什么，不被"黑盒"吓到   |
| **截图 + 反馈 = 最高效调试**    | 发现问题截图发给 Codex，比描述问题快10倍 |
| **Computer Use 是兜底方案** | 打包失败等复杂操作，让 AI 自己操作电脑解决  |

***

## 思考与延伸

1. **Vibe Coding 的门槛已经降到"会用电脑"**：文科生、零编程基础的人也能用 AI 开发 APP 并上架——这不是未来，是现在。
2. **"先视觉后代码"的开发范式**：传统开发是先写代码再看效果，Vibe Coding 2.0 是先看效果再写代码——这极大降低了沟通成本和返工率。
3. **Superpower 的产品经理角色**：AI 不只是执行者，还能帮你拆需求、做决策、出计划——这是 Vibe Coding 从"工具"到"搭档"的质变。
4. **Plan Mode 的透明度价值**：让用户看到 AI 的执行计划，避免"黑盒焦虑"——这是 AI Agent 产品设计的重要原则。
5. **插件生态是 Codex 的护城河**：Superpower + Computer Use + Chrome + Builds Apps 的组合，让 Codex 从"代码生成器"变成了"全栈开发团队"。

## 完整文稿

> 好了大家，今天这期是 Vibe Coding 2.0 的新手教程。去年我用 AI 开发了一个小猫禅式的 APP，有33万人看过。当时的 AI 虽然能够跑完 APP 开发，但过程当中经常会中断出错，而且要在 IDE 上面来回地折腾，操作非常麻烦。但不到一年的时间，Vibe Coding 这件事情变得不一样了——AI 不光会写代码，还会自己跑测试、自己截图找 bug、自己操作电脑帮你打包上架。所以这期我就带你手把手用 Codex 从开发到上架，更高效地跑完这一面流程。

> 这次我用的 AI Agent 工具是 Codex，它整体的页面设计是非常克制的，只有两个分页——左边是项目栏，右边是对话框。跟之前的 IDE 平台比没有密密麻麻的 MD 文件和代码窗口，对于完全不懂编程的人来说，比如像我这样一个文科生，就会友好很多。

> 在我们开始前需要做一件事情，就是去插件中心帮 Codex 加几个插件，它就能精通各种外接的技能。这次开发用到的插件清单：**Superpower**（产品经理角色）、**Computer Use**（AI 直接操作电脑）、**Chrome**（AI 在浏览器里操作）、**Builds Apps**（模拟器预览和测试）。

> 开始的第一步，先创建一个 APP 的项目文档。选择 o3-mini 模型、中度推理模式，使用下来速度和结果都 OK。

> 今天我要做一个 AI 美甲的项目。很多编程新手包括我一上来就会想要 AI 一句话就能帮我生成好一个 APP，结果生成的代码又多又长、额度消耗量又非常大，而且一直都没有结束，跑了半天你都不知道 AI 跑到哪里去了。所以我们在用 AI 开发的时候，需要非常专业的 AI 产品经理把控项目的开发进度——这就要用到 Codex 的插件 Superpower。

> 点击 Superpower，给它提示词："我要开发一款 AI 美甲的产品，并最终打包发布在 App Store 上面。"Superpower 会调用 Brainstorming 脑暴技能跟你有来有回地讨论，再用选择题的方式给你确认——核心目标、实现路径等。确认完之后它就输出一份非常完整的开发计划，包括流程设计、技术路线设计、核心功能、数据接口、测试验收等。你可以把它理解为"为你而生的开发地图"，只要顺着这个地图走就行了。

> 我非常建议在 AI 写任何一行代码之前，把前端页面视觉对齐。因为代码你看不懂，文字又容易有歧义，**图片就是最高效的对齐方式**。让它生成前端可视化展示，然后反复微调——logo 区域压低、留白增多、手的位置比例调整等。基于主页图生成其他接触页面，直接在页面上调整排版。

> 前端所有页面对齐之后，进入 coding 开发阶段。让它基于确认的前端图生成页面代码，等了44分钟生成了前端交互页面。用 Builds Apps 插件看模拟器效果，验证交互功能，发现问题截图发给 Codex 快速修改。

> 前端没问题后进入后端代码开发，同步开启 Plan Mode 了解 AI 每一步在做什么。AI 会给出完整任务计划，逐个实现上传图片、AI 生图、积分等功能。

> 开发到这个阶段大概率会遇到同一个问题——打包失败。用 Computer Use 插件让 AI 直接操作电脑，自己打开终端、运行命令、排查错误、修复问题，最终完成打包上架。
