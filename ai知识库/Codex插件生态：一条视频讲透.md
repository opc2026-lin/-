# Codex插件生态：一条视频讲透

> 作者：文走走的Ai分享
> 时长：约4分29秒
> 链接：[https://v.douyin.com/4Kot-tiaAuY/](https://v.douyin.com/4Kot-tiaAuY/)

***

## 核心观点

Codex 为什么好用？核心就一个字：**插件生态**。你的一个想法，Codex 可以借助各种插件把它变成文档、表格、PPT，甚至直接帮你处理实验数据、写网页、做设计方案。插件不是锦上添花，而是把 Codex 从一个"聊天工具"变成一个"全能工作台"的关键。

***

## 五大推荐插件

### 插件一：Documents（文档处理）

**适合场景**：写项目方案、汇报文档、技术文档等结构化长文。

**正确用法**：

1. **先开计划模式**（Plan Mode）—— 让 Codex 跟你一起把文档的骨架搭清楚
2. 再进入正式写作
3. 计划确认后再让它继续生成正文

**为什么先开计划模式很重要？**

> 如果没有计划模式，Codex 写出来的东西往往是一堆泛泛而谈的套话。先搭骨架再填肉，输出质量天差地别。

**进阶用法**：让 Codex 把本次用到的插件功能附在文档后面，比如：

* 文档生成
* 结构梳理
* 审阅批注
* 格式整理

> 这样不仅方案本身能交付，你也能清楚知道这个插件到底帮你完成了哪些具体工作。

***

### 插件二：Spreadsheets + Data Analytics（表格 + 数据分析）

**适合场景**：实验数据处理、项目实施数据、Excel/CSV 原始表格分析。

**正确用法（两步走）**：

**第一步：让 Codex 先读懂数据**

> 不要直接问"帮我分析一下"——先让它把数据结构看明白。

**第二步：再让它继续往下做**

* 统计不同时间段的数据变化
* 找出异常波动
* 比较不同实验组之间的差异
* 把关键指标做成图表

**典型应用场景**：

* 设备运行数据
* 实验测试数据
* 用户增长数据
* 销售转化数据

**工作流**：原始数据 → 清洗 → 分析 → 输出图表 → 整理成可汇报的数据结论

***

### 插件三：GitHub

**适合场景**：项目前期调研、找参考、做对比。

**正确用法**：

* 让 Codex 去 GitHub 上找同类型项目
* 看看别人是怎么拆功能、怎么处理配置、怎么写使用说明的
* 找某个 skill 或某类能力有没有实现

**核心价值**：

> 不是让你照抄别人的项目，而是让你在开始之前先看到成熟项目是怎么做的。

**典型工作流**：

```
项目启动前 → GitHub 找参考 → 做对比 → Codex 整理成自己的实现方案
```

***

### 插件四：Creative Production（商业创意工作台）

**适合场景**：产品/服务/门店/活动/广告创意的直接落地实现。特别适合团队工作及企业落地。

**正确用法示例**：做一个线下咖啡产品上市的广告创意

**指令结构**：

```
@creative production
目标：吸引城市白领
渠道：短视频平台和线下海报
风格：高级但不要冷
先探索创意方向
```

**Codex 的工作流程**：

1. 给你一个 Creative Production 的探索入口
2. 你自己选择最适合的创意路径
3. 完成视觉设计（如12个具有明确主题的视觉样板）
4. 每个 prompt 约束成单张照片风格参考图
5. 输出12张独立参考图
6. 汇总到 Moodboard 评审版
7. 你选择满意的图片，让它继续完成海报设计

**示例主题**：城市通勤、门店入口、手作细节、午后 reset、白领短暂停留、夜色暖光等。

***

### 插件五：Computer Use（计算机使用）—— 颠覆性功能

**核心能力**：让 Codex 通过截图 + macOS 辅助功能读取屏幕上的 APP，然后像人一样点击、输入、滚动、拖拽、按快捷键。

> 说白了：让模型通过用户界面操作软件。

**真实案例**：地图软件操作项目

* 地图软件没有可介入的 API
* 调用 Computer Use 功能
* 把地图软件从"只能人工看的界面"变成"可以参与核对的数据现场"
* Codex 完成了：数据整理 → 地图标记 → 结合标记结果进行数据诊断分析 → 形成项目方案正式文件

**效率对比**：

* 人工做：一个星期都不一定能搞定
* Codex + Computer Use：一个小时解决

**缺点**：额度消耗很快。

***

## 插件使用心法

| **插件**                        | **核心心法**                |
| ----------------------------- | ----------------------- |
| Documents                     | 先计划模式搭骨架，再生成正文          |
| Spreadsheets + Data Analytics | 先让 Codex 读懂数据，再提分析需求    |
| GitHub                        | 找参考 → 做对比 → 整理成自己的方案    |
| Creative Production           | 先探索方向 → 选路径 → 再细化执行     |
| Computer Use                  | 没有 API 时的终极兜底方案，但注意额度消耗 |

***

## 思考与延伸

1. **计划模式是 Documents 的灵魂**：很多人直接让 AI 写文档，结果得到一堆套话。先搭骨架再填肉，这是专业写作者的基本功，也是 AI 写高质量文档的前提。
2. **"先读懂数据"的数据分析原则**：不要直接问"帮我分析"——让 AI 先看数据结构，再提具体问题。这和人做数据分析的流程一致。
3. **GitHub 插件的价值被低估**：不是抄代码，而是看成熟项目的功能拆分、配置处理、文档写法——这是项目启动前最有价值的调研步骤。
4. **Creative Production 的创意工作流**：从"探索方向"到"选路径"再到"细化执行"——这个三段式工作流适用于几乎所有创意类任务。
5. **Computer Use 的边界与成本**：它是没有 API 时的终极兜底方案，但额度消耗快。应该作为"最后手段"而不是"首选方案"。

> **第一个，Documents 文档处理。** 比如你要写一份项目方案，提前打开计划模式，然后直接告诉 Codex：帮我写一份某某治理项目的方案，面向甲方汇报。记得最后把插件功能附上。计划模式其实特别重要，Codex 会跟你一起把文档的骨架搭清楚，再进入正式写作。等计划确认之后，你再让它继续生成正文，它写出来的东西就不会是一堆泛泛而谈的套话。最后你还可以让它把本次用到的插件功能附在文档后面，比如用了文档生成、结构梳理、审阅批注、格式整理这些能力。这样不仅方案本身能交付，你也能清楚知道这个插件到底帮你完成了哪些具体工作。

> **第二个，Spreadsheets 和 Data Analytics。** 这是两个插件，一个是表格插件，一个是数据分析插件，我一般都一起用。比如你的试验数据、项目的各类实施数据，或者是 Excel/CSV 里整理出来的一堆原始表格，都可以交给它来处理。我一般不会直接问它"帮我分析一下"，我会先让它读懂数据，等它把数据结构看明白之后，我再让它继续往下做。比如帮我统计不同时间段的数据变化、找出异常波动、比较不同实验组之间的差异、或者把关键指标做成图表。如果你做的是项目类工作，这个就特别实用。比如设备运行、实验测试、用户增长、销售转化，这些数据本身可能很乱，但是通过这两个插件配合，就可以先清洗、再分析、再输出图表，最后整理成一份能汇报的数据结论。

> **第三个，GitHub。** 在你准备做一个项目之前，可以先让 Codex 去 GitHub 上找参考。比如你想做一个自动化工具，可以让它帮你找同类型项目，看看别人是怎么拆功能、怎么处理配置、怎么写使用说明的。甚至你想找某个 skill 或者想知道某类能力有没有实现，也可以先去 GitHub 上找。它的价值不是让你照抄别人的项目，而是让你在开始之前先看到成熟项目是怎么做的。所以 GitHub 这个插件我一般会用在项目前期，先找参考、再做对比、再让 Codex 帮我整理成自己的实现方案。

> **第四个，Creative Production。** 你可以把它理解成商业创意的工作台。它不只是生成一张好看的图，而是把产品、服务、门店、活动、广告创意直接落地实现。很适合团队工作及企业落地。比如你可以 @creative production 做一个线下咖啡产品上市的广告创意，目标是吸引城市白领，渠道是短视频平台和线下海报，风格要高级但不要冷，先探索创意方向。Codex 会先给你一个 creative production 的探索入口，你自己选择最适合这次上市的创意路径。我这里是先让它完成视觉设计，它做出12个具有明确主题的视觉样板，包括城市通勤、门店入口、手作细节、午后 reset、白领短暂停留、夜色暖光等。而且每个 prompt 都会约束成单张照片风格参考图，避免拼贴文字 logo 促销和网红甜品风。最后输出的结果是12张独立的参考图，然后会汇总到 moodboard 评审版让你审阅。你可以选择自己满意的图片，让它继续完成后面的海报设计。

> **第五个，Computer Use。** 这个插件我觉得是颠覆性的功能。让 Codex 通过截图加 macOS 的辅助功能读取屏幕上的 APP，然后像人一样点击、输入、滚动、拖拽、按快捷键。说白了就是：让模型通过用户界面操作软件。我用这个功能干了非常多的事情。比如我前段时间做了一个项目，主要涉及到对地图软件的操作，没有可以介入的 API，所以我就调用了 Codex 的 computer use 功能。简单讲就是把地图软件从一个只能人工看的界面，变成了我可以参与核对的数据现场。它帮我们完成了数据整理到地图标记、结合标记结果进行数据诊断分析、最后形成项目方案正式文件的所有工作。我自己干一个星期都不一定能搞定，Codex 一个小时解决。缺点就是额度消耗很快。
