# 装过上百个Skill后，推荐这16个

> 作者：起哥的AI实战
> 时长：约3分12秒
> 链接：[https://v.douyin.com/eshTkvcVSk4/](https://v.douyin.com/eshTkvcVSk4/)

***

## 核心观点

作者在尝试过数百个 AI Skill 后，精选出 16 个最常用的 Skill，按功能分为六大类：**联网搜索、内容生成、开发工具、测试优化、运维工具、前端设计**。核心思路是：让 AI 工具链形成闭环——从信息搜集、内容创作、开发落地到质量检测，每个环节都有对应的最佳 Skill。

***

## 16 个推荐 Skill 分类整理

### 一、联网搜索（3个）

| **Skill**                    | **功能**                                                |
| ---------------------------- | ----------------------------------------------------- |
| **AgentReach**               | 搜索国内外 17 个平台内容（含主流自媒体平台）                              |
| **Last24HoursNews**          | 搜索最近发生的新闻（AI相关、时事相关），充当信息采集员                          |
| **Suppresscribe / WebFetch** | 搜索并抓取网页内容，输出纯净 Markdown 格式，剔除杂音，节省 token 并提高 AI 识别准确率 |

### 二、内容生成（2个）

| **Skill** | **功能**               |
| --------- | -------------------- |
| **开头优化器** | 优化短视频开头 5 秒的表述，提升完播率 |
| **封面生成器** | 自动生成视频封面             |

### 三、开发工具 — Superpowers 系列（5个）

| **Skill**                 | **功能**                                    |
| ------------------------- | ----------------------------------------- |
| **Brainstorming**         | 当想法模糊时，通过多轮对话将模糊想法转化为清晰需求                 |
| **Writing Plan**          | 将清晰需求落地为可执行的开发计划                          |
| **Markdown Converter**    | 将 PDF、PPT 等多格式文件转换为纯净 Markdown，提升 AI 解析效率 |
| **Skill Creator（元Skill）** | 创建其他 Skill 的"原 Skill"，用于生成自定义 Skill       |
| **Videocard**             | 视频剪辑助手：自动去除气口、重复表述、停顿，自动加字幕               |

### 四、测试优化（2个）

| **Skill**                 | **功能**                 |
| ------------------------- | ---------------------- |
| **Content Risk Detector** | 拍摄前检测脚本是否违反平台规则，避免限流风险 |
| **Humanizer**             | 内容去 AI 化，让表达更自然、更像真人说话 |

### 五、运维工具（1个）

| **Skill**   | **功能**        |
| ----------- | ------------- |
| **ZeroSSL** | 自动生成 HTTPS 证书 |

### 六、前端设计（3个）

| **Skill**           | **功能**                                         |
| ------------------- | ---------------------------------------------- |
| **Frontend Design** | 保证 UI 设计一致性，适合内部工具                             |
| **UI/UX Pro Max**   | 高质量前端设计 Skill                                  |
| **Impactable**      | 对外产品设计：支持 Level 模式，可对页面不满意处做局部修改，提供 3 个版本可视化操作 |

***

## 思考与延伸

1. **你的 AI 工具链完整吗？** 作者推荐的 16 个 Skill 覆盖了信息采集→内容创作→开发实现→质量检测→部署上线的全流程。对照自己的工作流，你缺了哪个环节？
2. **元 Skill 的价值**：Skill Creator（原 Skill）是最值得关注的一个——掌握了"创造工具的工具"，就能不断扩展自己的能力边界。
3. **去 AI 化的重要性**：很多人只关注 AI 生成内容，却忽略了 Humanizer 和 Content Risk Detector 这两个"把关型"工具，它们往往决定了最终输出能否真正落地。
4. **前端设计的层次感**：内部工具 vs 对外产品，对 UI 的要求完全不同。Impactable 的 Level 模式（局部修改 + 多版本可视化）代表了新一代前端工具的方向。

## 完整文稿

> 大半年尝试了数百个 Skill，最终只留下 16 个最常用的。
>
> 首先是三个**联网搜索**的 Skill：
>
> * 第一个是 **AgentReach**，它可以搜索国内外 17 个平台的内容，国内主流的自媒体平台都可以在上面搜索到。
> * 第二个是 **Last24HoursNews**，如果你想搜索最近发生的一些新闻，比如 AI 相关的或者时事相关的，都可以通过这个 Skill 帮你搜集最新的信息，是一个很好的信息采集员。
> * 第三个是 **Suppresscribe**（WebFetch），它可以帮你搜索和抓取网页里的内容，它有一个很大的特点就是抓取的内容是非常纯净的 Markdown 格式，把其他的一些杂音全部都剔除掉了。AI 对于 Markdown 的识别是比较擅长的，不仅能够节省 token 还能提高识别的准确率。
>
> 接下来是两个**内容生成**的 Skill：
>
> * 第一个是 **开头优化器**，我自己做自媒体的短视频，开头 5 秒其实是非常重要的，我是通过这个开头优化器帮我去优化逐字稿中的开头的表述。
> * 第二个是 **封面生成器**，我自己大部分的视频封面都是通过这个 Skill 来生成的。
>
> 接下来是五个**开发工具**，首先是 **Brainstorming** 和 **Writing Plan**，这两个 Skill 我是不相搭配着用的，他们都是 Superpowers 下面的子 Skill。当我脑子里有一个模糊的想法的时候，我首先会通过 Brainstorming 跟我多沟通几轮，把我一个模糊的想法转换成一个清晰的需求，然后通过 Writing Plans 把这个需求落地成可以执行的计划。
>
> 第二个是 **Markdown**，它可以把很多非 Markdown 格式的文件替你转换成 Markdown 格式。比如你给 AI 很多像 PDF、PPT 或是多种格式的文档，它可能解析起来很慢而且其中可能包含很多无用的信息，通过这个工具给你转换一下就是比较干净的 Markdown 格式。
>
> 第三个是 **Skill Creator**，它被称作是"原 Skill"，就是帮你创建 Skill 的 Skill。我自己做很多 Skill 也是通过它来生成的。
>
> 最后一个是 **Videocard**，做视频剪辑的。我自己视频拍摄出来之后我都会扔给它，让它帮我进行初剪，比如气口、重复的表述、还有一些停顿全部都给自动处理了，然后自动给我打上字幕，这样就会提升我剪辑视频的效率。
>
> 接下来是两个**测试优化**的 Skill：
>
> * 第一个是 **Content Risk Detector**，我在拍摄视频之前都是通过这个 Skill 先把我的逐字稿给检测一遍，看是否存在违反平台规则的地方。因为如果辛辛苦苦拍出来视频发布的时候被限流，那肯定是一个非常大的浪费，我都是通过这个 Skill 来帮我把关的。
> * 第二个是 **Humanizer**，它是专门帮你把内容去 AI 化的。我自己做的逐字稿基本上都会先通过它来帮我去变 AI 化，这样听起来会更自然，更像是在说人话。
>
> 然后是一个**运维相关**的 Skill 叫 **ZeroSSL**，它会帮我自动去生成 HTTPS 证书。
>
> 最后是三个**前端设计**的 Skill：**Frontend Design**、**UI/UX Pro Max** 相信很多人都耳熟能详，我自己做一些内部工具的时候都是通过它来帮我保证 UI 设计的一致性的。但是如果当需要设计一个对外的产品，那对于界面的要求会比较高，我就会用 **Impactable**。它有一个特色的功能就是有 Level 的模式，你可以对页面中你不满意的地方做局部的修改，它会给你提供三个版本而且是可视化的操作，这样用起来就会非常的方便。
>
> 以上就是我最常用的 16 个 Skill，如果你有一些好用的 Skill，欢迎在评论区分享。我是起哥，请用 AI 前线生产力。
