# Open Design AI技能探究

> 作者：全栈观察员
> 时长：约4分24秒
> 链接：[https://v.douyin.com/JdZvUkIMCSU/](https://v.douyin.com/JdZvUkIMCSU/)

***

## 核心观点

Open Design 是一个做设计的开源项目，上线不到一周获得 2 万 star，热度碾压市面上 90% 的 AI 工具。它之所以爆火，答案藏在一个词里：**操作系统**。

市面上所有 AI 设计工具都在拼同一件事：谁的模型更强、谁出图更好看。Open Design 没走这条路，它问了一个别人都没问过的问题：

> **AI 的设计能力，能不能拆开、组装、分享？**

这就是它 5 天长 2 万 star 的真正原因。

***

## 三层架构

### 第一层：技能即文件（Skills as .md）

在 Open Design 里，每项设计能力就是一个 `Skill.md` 文件。写一个 Markdown 文件，AI 就学会了一项技能。

* 不是"下载→安装→重启"的套路
* 更像是**给 AI 写教材**
* 259 个技能 = 259 份教材
* 你想要一个能做数据仪表盘的技能？写个 Skill.md 把步骤写清楚，AI 读完就会了

### 第二层：管道式工作流（Pipelined Workflow）

```
Plan（规划）→ Generate（生成）→ Critique（评审）→ Handoff（交付）
```

四个阶段像流水线一样串联，**每一步都能单独配置**：

* 可以把 Critique 阶段关掉，让 AI 直接出图
* 可以插入一个自定义的评审逻辑
* 相当于你在**自定义 AI 设计的思考过程**

> 这个能力，其他工具根本没有。

### 第三层：多 Agent 接口 + AMR 模型路由

17 个 agent 接口，主流模型全部兼容：

* Claude / Claude Code
* Cursor
* Gemini / Gemini CLI
* Codex

**AMR（Auto Model Routing）** 更进一步：一次充值，每一步自动选最合适的模型——

| **阶段** | **推荐模型** |
| ------ | -------- |
| 规划阶段   | 用便宜的     |
| 生成阶段   | 用强的      |
| 评审阶段   | 用快的      |
| 谁合适用谁  | 不浪费一分钱   |

***

## 142 个可移植设计系统

内置 142 个设计系统，不是随便堆在一起的，而是**可移植的品牌基因包**：

* 把一个设计系统从一个项目搬到另一个项目
* AI 自动继承配色、间距、组件风格

**这意味着什么？**

> 你团队的设计规范不再是 PDF 文档，而是 AI 能直接读懂执行的配置文件。
>
> 设计系统从"给人看的规范"变成了"给机器执行的程序"。

***

## 插件生态：按工作流阶段分类

| **类别**     | **作用** |
| ---------- | ------ |
| **Create** | 创建     |
| **Export** | 导出     |
| **Import** | 导入     |
| **Share**  | 分享     |
| **Deploy** | 部署     |
| **Refine** | 精修     |
| **Extend** | 扩展     |

> 注意：这个分类方式**不是按功能分的，而是按工作流的阶段分的**。创建和导出是不同的事，导入和分享也是不同的事。
>
> 每个插件就是管道里的一个**原子操作**，可以自由组合——这像 Unix 管道哲学：**每个工具只做一件事，但拼起来能解决复杂问题**。

***

## Open Design vs 封闭系统（对比）

| **维度** | **封闭系统（如 Figma AI）** | **Open Design** |
| ------ | -------------------- | --------------- |
| 模型     | 模型锁死                 | 可以随便换           |
| 能力     | 能力固定                 | 技能可以自己写         |
| 工作流    | 工作流不可调               | 工作流可以自己拼        |
| 本质     | 成品家电                 | 乐高积木            |

> **你买的不只是功能，是组装能力。**
>
> 这就是为什么它叫"AI 设计**操作系统**"而不是"AI 设计工具"。

***

## 对普通开发者的意义

**你不需要会设计，也不需要懂模型调参。**

你需要做的只是：**写清楚你想要什么**。

* Skill.md 的格式就是 Markdown
* 你会写文档，就会写技能
* 写好之后丢给 Open Design，它会自动：规划 → 生成 → 评审 → 交付
* 导出时 HTML、PDF、PPTX、MP4 全都能出

**从想法到成品，AI 帮你把中间的设计环节全部流水化了。**

***

## 真正的价值

Open Design 真正的价值不是 259 个技能，也不是 142 个设计系统，而是它**证明了**一件事：

> 当 AI 的能力被拆成最小的独立单元，工作流可以被自由编排，模型可以随时替换——AI 设计就不再是某个公司的特权，而是任何人都能组装的系统。

**这个架构思路，才是它 5 天 2 万 star 的真正原因。**

***

## 思考与延伸

1. **操作系统思维**：作者用"操作系统"来定义 Open Design，本质是"能力原子化 + 自由组合"。这和 Unix 管道的哲学一脉相承。
2. **设计系统的范式转变**：从"给人看的规范文档"到"给机器执行的设计程序"，这是设计系统从 PDF 时代进入 AI 时代的标志。
3. **AMR 模型路由的价值**：不是用一个模型做所有事，而是在正确的地方用正确的模型——这将催生新的 AI 工作流优化思路。
4. **Skill.md 作为技能标准**：把技能写成 Markdown 文件，意味着任何人可以分享、复用、改进 AI 技能——这是一种去中心化的 AI 能力共享机制。
5. **对 Vibe Coding 的影响**：Open Design 证明了"设计"也可以被 Vibe Coding 化——设计师不再是必需，但设计能力可以通过组合 AI 单元获得。
