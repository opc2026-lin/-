3. 天气匹配逻辑（必须使用修正版）
3.1 核心原则
后续所有训练 / 预测 / 验证版本，默认都用天气修正版，不能回退到旧版。

3.2 地区标准化规则
主档案区县标准化
使用：

Python

normalize_region_name()
作用：

去空格
去掉末尾行政后缀：
区
县
市
例如：

鼓楼区 → 鼓楼
连江县 → 连江
天气地区标准化
天气的 region 先拆成：

市
区县
然后区县也走：

Python

normalize_region_name()
3.3 匹配优先级
第一步：严格匹配
按：

Python

["所在市_norm", "所在区_norm", "datetime"]
第二步：城市级聚合回补
如果区县小时匹配不到：

数值列：按同市同小时求均值
类别列：按同市同小时取众数
不能再用：

Python

groupby(...).first()
因为这会造成随机区县污染。

3.4 天气匹配层级字段
必须保留字段：

Python

weather_match_level
可能值：

district_exact
city_agg
用途：

训练后分析天气匹配质量
验证时可按天气匹配层级分组评估
3.5 训练时天气完整性
训练阶段：

若某天任一小时关键天气缺失
可整天剔除
关键字段至少：

Python

通常会生成：

Python
