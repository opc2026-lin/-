# 视频笔记生成器

当用户发送一个视频链接时，自动完成以下流程。

## 前置要求

Obsidian 需安装 **Surfing** 插件（设置 → 第三方插件 → 浏览 → 搜 "Surfing"）。
Surfing 让 Obsidian 支持内嵌网页，视频可以在笔记内直接播放，不用跳转浏览器。

## 执行流程

### Step 1: 获取视频信息

使用 `WebFetch` 抓取视频页面，提取：
- 标题
- 作者/UP主
- 视频 ID（从 URL 中提取）
- 时长（如有）

若 WebFetch 无法直接抓取（如抖音域名被屏蔽），改用 **WebSearch** 搜索视频标题、作者和关键内容，从搜索结果和转载文章中间接获取视频信息和核心内容。

### Step 2: 生成笔记内容

基于获取到的视频信息（标题、描述、字幕/逐字稿如有），AI 生成以下内容：

#### 核心观点与总结
- **核心论点**：一段话概括视频的核心主张
- **核心要点**：分点列出关键知识点
- **行动建议**（如适用）：可操作的具体建议

#### 联系实际的思考题
- 生成 3 个思考题，设计原则：
  - 第 1 题：联系用户自身经历
  - 第 2 题：假设一个应用场景
  - 第 3 题：延伸思考/批判性反思

#### 视频逐字稿
- **优先使用本地转录**：运行 `python3 .claudian/skills/video-note/transcribe.py "<视频URL>"` 
  - 抖音：解析 share 页 JSON → 管道流式获取媒体 → ffmpeg 丢弃视频帧只留音频 → Whisper 转录，**全程不存视频**
  - B站/YouTube：yt-dlp `-f bestaudio` 只拉音频流 → Whisper 转录
  - 耗时约 2-5 分钟，转录质量高
- 如果本地转录失败（如首次需要下载模型或网络问题）：用 WebSearch 搜索「视频标题 + 文字版」获取转载内容
- **绝对不要留空**，有逐字稿放逐字稿，没有就放搜索到的最完整版本

### Step 3: 保存文件

按以下规则保存到 `D:\opc-ai知识库\收件箱\`：

**文件名格式**：`视频笔记_{简短标签}_{视频ID}.md`

- `简短标签`：从标题中提取 3-6 个中文字作为关键词标签，移除特殊字符
- `视频ID`：从 URL 中提取
- 若文件名已存在，追加 `_2` 等序号

**内容模板**：

```markdown
# {视频标题}

## 视频信息

<iframe src="{嵌入URL}" width="100%" height="480" frameborder="0" allowfullscreen style="border-radius:8px;margin-bottom:12px"></iframe>

| 项目 | 内容 |
|------|------|
| **标题** | {标题} |
| **作者** | {作者} |
| **视频ID** | {视频ID} |
| **时长** | {时长} |
| **链接** | [{原始URL}]({原始URL}) |
| **平台** | {抖音/B站/YouTube} |

---

## 一、核心观点与总结

### 核心论点

{一段话概括视频的核心主张}

### 核心要点

1. **{要点1标题}**：{说明}
2. **{要点2标题}**：{说明}
3. **{要点3标题}**：{说明}

---

## 二、联系实际的思考题

1. **{思考题1}**

2. **{思考题2}**

3. **{思考题3}**

---

## 三、视频逐字稿

{逐字稿内容，或 "逐字稿待补充，请访问原视频查看: {URL}"}

---

*笔记由 AI 自动生成，内容基于视频信息提取，仅供参考。*
```

### 各平台内嵌 URL 格式

| 平台 | 内嵌 URL 模板 | 说明 |
|------|-------------|------|
| B站 | `//player.bilibili.com/player.html?bvid={bvid}&page=1&autoplay=0` | 从 `b23.tv` 短链或 `bilibili.com/video/{bvid}` 提取 bvid |
| YouTube | `https://www.youtube.com/embed/{video_id}` | 从 `youtube.com/watch?v=` 或 `youtu.be/` 提取 video_id |
| 抖音 | `https://open.douyin.com/player/video?vid={video_id}&autoplay=0` | 从短链 `v.douyin.com/xxx` 跟随重定向获取真实 video_id。需要加 `referrerpolicy="unsafe-url"` |

**注意**：内嵌 `<iframe>` 在 Obsidian 阅读模式下直接渲染。如果使用实时预览模式，需要 **Surfing** 插件。如果 iframe 不显示，点击表格中的 `链接` 行可在浏览器打开。

### Step 4: 确认输出

完成后告知用户：
- 文件保存位置：`收件箱/视频笔记_xxx.md`
- 关键信息摘要
- 视频已内嵌，可在 Obsidian 内直接观看

---

## 每月整理

每月 1 日自动执行收件箱整理：

1. 列出 `收件箱/` 下所有 `视频笔记_*.md` 文件
2. 按视频分类统计（作者、平台、主题聚类）
3. 生成月度整理报告：`收件箱/月度整理_{YYYY-MM}.md`
4. 按照 Ingest 路由表，询问用户是否将已看过的视频笔记归档到对应知识库目录
