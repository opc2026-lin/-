# 滚动撮合复盘标准模板

## 目的

这份模板只复盘一件事：

- **中长期底仓之上，连续滚动撮合这一步做得对不对**

不再混淆：

- 中长期持仓曲线
- 用户最终实际用电曲线
- 滚动撮合单次申报动作

复盘核心改为：

1. 净敞口判断是否正确
2. 报量强度是否合理
3. 报价是否到位
4. 成交后是否有效修复净敞口

---

## 使用顺序

每个目标日按下面顺序复盘：

1. 填写基础信息
2. 填写 24 时段逐小时数据
3. 给每个时段打结论标签
4. 汇总当日问题
5. 沉淀出下一次申报规则

---

## 二、24时段逐小时复盘主表

字段说明：

- `预测用电`：当时交易前使用的预测值
- `中长期持仓`：年度/月度/周及其他中长期底仓合计
- `净敞口`：`预测用电 - 中长期持仓`
- `申报量`：本次滚动撮合实际申报量
- `报量比例`：`申报量 / 净敞口`
- `申报价`：本次实际申报价
- `市场成交参考价`：当时能观察到的成交价或事后复盘成交价
- `成交量`：实际成交量
- `修复比例`：`成交量 / 净敞口`
- `时段分层`：低价值 / 一般 / 关键
- `动作类型`：试探补仓 / 正常补仓 / 抢量补仓 / 防高价少报 / 判断不买
- `结果分类`：方向对量少 / 方向对价低 / 方向对执行可 / 方向错不该买 / 方向错应更积极 / 试探未成可接受

## 三、固定判定规则

### 1. 时段分层

```text
低价值时段：
- 本来就不缺
- 或缺口小
- 或不值得高价抢

一般时段：
- 有缺口
- 但不是核心风险时段

关键时段：
- 缺口大
- 且晚峰/高价/保供压力大
```

### 2. 报量比例判定

```text
0 ~ 30%：试探型
30% ~ 70%：中性型
70% ~ 100%：进攻型
>100%：激进型，必须写原因
```

### 3. 价差判定

公式：

- `价差 = 申报价 - 市场成交参考价`

解释：

```text
价差 >= 0：价格达到成交要求
-1 ~ 0：只差临门一脚
-3 ~ -1：价格偏低
<-3：明显不是成交价
```

### 4. 修复比例判定

公式：

- `修复比例 = 成交量 / 净敞口`

解释：

```text
<30%：修复弱
30% ~ 70%：修复一般
>70%：有效修复
>100%：可能过补
```

### 5. 没成交性质

只允许填两类：

```text
可接受：
- 本来就是试探单
- 或该时段不是关键时段

不可接受：
- 本来必须成交
- 但因为量/价失误没拿到
```

## 四、结果分类枚举

每小时结论只从下面选：

```text
方向对，量少了
方向对，价低了
方向对，执行可以
方向错，不该买
方向错，应该更积极
试探未成，可接受
成交了，但买多了
成交了，但不是关键时段
```

## 五、当日汇总

```text
1. 今天最关键的错因是什么？
- 预测错
- 净敞口判断错
- 报量偏保守
- 报价偏保守
- 关键时段不够激进
- 非关键时段反而买多

2. 今天最值得保留的动作是什么？

3. 今天最不该重复的动作是什么？

4. 如果同样的盘面再来一次，哪些时段会改？

5. 明天开始固定执行的规则是什么？
```

## 六、规则沉淀页

每次复盘最后，只允许沉淀成“可执行规则”，不要写空话。

格式如下：

```text
规则1：
适用条件：
执行动作：

规则2：
适用条件：
执行动作：

规则3：
适用条件：
执行动作：
```

示例：

```text
规则1：
适用条件：17-20时净敞口 > 0.8 MWh，且当日高温
执行动作：第一笔不再试探，直接按关键时段成交价带报价

规则2：
适用条件：1-8时净敞口 <= 0
执行动作：原则上不买，不为抄底而抄底

规则3：
适用条件：申报价距离参考成交价超过 -3 元/MWh
执行动作：标记为“价低失误”，下次不再重复
```

---

## 七、最简复盘口径

如果时间不够，至少复盘这 5 个问题：

1. 哪几个小时本来就不该买？
2. 哪几个小时本来必须买？
3. 哪几个小时是价格低了？
4. 哪几个小时是报量少了？
5. 哪几个小时成交后真正修复了风险？

---

## 八、建议

以后每次复盘，不要只看：

- 成没成交
- 平均成交价

要优先看：

- 关键时段有没有修掉净敞口
- 非关键时段有没有买冤枉量
- 没成交是策略允许，还是执行失误

一句话：

- **滚动撮合复盘，复的是“剩余风险修复能力”，不是单纯成交结果。**
