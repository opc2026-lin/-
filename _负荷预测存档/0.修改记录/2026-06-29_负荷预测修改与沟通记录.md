## 我这边给出的主要判断

### 1. 图片功能不会自动提高精度

结论：

- 当前 `v3` 代码里并没有真正把图片数据喂进模型
- 所以“加入图片”本身不会直接让预测更准

原因：

- 代码读取的仍然是 Excel 天气文件
- 没有 OCR、没有图像特征提取、没有图像模型输入

### 2. `total_load` 的作用

结论：

- `total_load` 是“还原后的真实总用电需求”

公式：

```python
total_load = net_load + pv_est
```

解释：

- `net_load` 是表计看到的净负荷
- `pv_est` 是估算的光伏发电
- 对有光伏的用户，中午表上负荷会被光伏压低
- 如果模型直接学净负荷，会把光伏影响误当成用电行为

所以预测链路应当是：

```python
先预测 total_load
再减去 pv_est
得到 pred_net_load
```

### 3. 为什么原来偏差会大

我给出的核心结论：

1. 验证脚本不是真回测
2. 训练里部分指标是训练集自评
3. `total_load` 目标和历史特征口径不一致
4. 预测骨架过度依赖天气表完整性
5. 天气文件重复时没有明确“最新优先”规则
