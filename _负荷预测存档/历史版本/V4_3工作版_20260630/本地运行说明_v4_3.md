# 本地运行说明 v4.3

## 目标

以后你在本地直接运行，不需要我帮你每次手工跑。

- 输入目录：`D:\亿云能源科技售电交易数据库\1-1负荷预测输入`
- 输出目录：`D:\亿云能源科技售电交易数据库\1-2负荷预测输出`

## 最简单用法

### 方式 1：双击批处理

双击：

- `一键运行负荷预测_v4_3.bat`
- `手动日期运行_v4_3.bat`

然后手工输入：

1. 运行模式
2. 目标日期
3. 连续天数

说明：
- 如果你只是想手动输入日期后直接跑，优先双击 `手动日期运行_v4_3.bat`
- 不建议直接双击 `.py` 文件

### 方式 2：运行 Python 交互脚本

```bash
python 05_auto_run_v4_3.py
```

它会在终端里让你手工输入：

1. 目标日期
2. 连续天数

注意：
- 这里已经按你的要求改成“手工输入日期”
- 不再自动识别日期

## 命令行用法

### 全流程

```bash
python run_forecast_v4_3.py --mode all --start-date 2026-07-03 --days 1
```

### 只预测

```bash
python run_forecast_v4_3.py --mode predict --start-date 2026-07-03 --days 1
```

### 只核验

```bash
python run_forecast_v4_3.py --mode verify --start-date 2026-07-03 --days 1
```

### 交互式输入日期

```bash
python 05_auto_run_v4_3.py --mode all
```

### 直接在交互脚本里带日期

```bash
python 05_auto_run_v4_3.py --mode all --start-date 2026-07-03 --days 1
```

## 运行模式

- `all`：训练 + 预测 + 回测 + 核验
- `train`：只训练
- `predict`：只预测
- `validate`：只回测
- `verify`：只核验

## 核验要求

以后凡是改了负荷预测代码，至少要完成：

1. `py_compile`
2. 目标日期预测重跑
3. `04_verify_v4_3.py` 核验通过

## 注意

1. 真实回测依赖真实负荷数据  
   如果目标日期的真实负荷还没放进输入目录，回测会提示数据未到。

2. 预测天气采用最新文件优先规则  
   当前逻辑会优先使用最新批次天气文件。

3. 测试目录只保留一个主版本  
   老版本都会移到历史版本目录。
